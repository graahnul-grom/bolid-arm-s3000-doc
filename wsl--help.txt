PS C:\> wsl --help

Использование: wsl.exe [Argument] [Options...] [CommandLine]

Аргументы для выполнения двоичных файлов Linux:

    Если командная строка не указана, wsl.exe запускает стандартную оболочку.

    --exec, -e <CommandLine>
        Выполнение указанной команды без использования стандартной оболочки Linux.

    --
        Передача оставшейся командной строки как есть.

Параметры:
    --cd <Directory>
        Настройка указанного каталога в качестве текущего рабочего каталога.
        Если применяется символ "~", будет использоваться домашний путь пользователя Linux. Если путь начинается
        с символа "/", он будет интерпретирован как абсолютный путь Linux.
        В противном случае значение должно быть абсолютным путем Windows.

    --distribution, -d <Distro>
        Запуск указанного распределения.

    --user, -u <UserName>
        Запуск от имени указанного пользователя.

Аргументы для управления подсистемой Windows для Linux:

    --help
        Отображение сведений об использовании.

    --install [Параметры]
        Установка дополнительных распределений подсистемы Windows для Linux.
        Для получения списка допустимых распределений используйте команду wsl --list --online.

        Параметры:
            --distribution, -d [Аргумент]
                Загрузка и установка распределения по имени.

                Аргументы:
                    Допустимое имя распределения (без учета регистра).

                Примеры:
                    wsl --install -d Ubuntu
                    wsl --install --distribution Debian

    --set-default-version <Версия>
        Изменяет версию для установки по умолчанию для новых распределений.

    --shutdown
        Немедленно завершает работу всех распределений и WSL 2
 упрощенного служебного виртуального компьютера.

    --status
        Показывает состояние подсистемы Windows для Linux.

    --update [Параметры]
        Если параметры не заданы, ядро WSL 2 будет обновлено
        до последней версии.

        Параметры:
            --rollback
                Восстановить предыдущую версию ядра WSL 2.

Аргументы для управления распределениями в подсистеме Windows для Linux:

    --export<Distro> <Имя файла>
        Экспортирует распределение в tar-файл.
        Для стандартной выгрузки имя файла может быть «-».

    --import <Distro> <Расположение установки> <Имя файла> [Параметры]
        Импортирует указанный tar-файл как новое распределение.
        Для стандартной загрузки имя файла может быть «-».

        Параметры:
            --version <Версия>
                Указывает версию для использования для нового распределения.

    --list, -l [Параметры]
        Выдает список распределений.

        Параметры:
            --all
                Список всех распределений, включая распределения
                в процессе установки или удаления.

            --running
                Список работающих в настоящий момент распределений.

            --quiet, -q
                Показать только имена распределений.

            --verbose, -v
                Показать подробную информацию обо всех распределениях.

            --online, -o
                Отображение списка доступных распределений для установки с помощью команды wsl --install.

    --set-default, -s <Distro>
        Задает распределение по умолчанию.

    --set-version <Distro> <Version>
        Изменяет версию указанного распределения.

    --terminate, -t <Distro>
        Завершает указанное распределение.

    --unregister <Distro>
        Отменяет регистрацию распределения и удаляет корневую файловую систему.
